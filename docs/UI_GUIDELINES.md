# UI Guidelines

## 기본 원칙
- 모바일 퍼스트
- 카드 기반 레이아웃
- 시각적 장식보다 정보 전달 우선

## 금지 사항
- 페이지 전용 버튼 스타일 생성 금지
- 임의 애니메이션 추가 금지

## 사용자 피드백 및 에러 처리 (User Feedback)
- **성공/실패 알림**: 모든 Form Action 결과는 `sonner` 또는 `toast`를 통해 사용자에게 피드백을 제공해야 한다.
- **로딩 상태**: 데이터 로딩 중에는 반드시 `Skeleton` 또는 `Spinner`를 표시하여 멈춘 느낌을 주지 않아야 한다.
- **폼 에러**: 입력 필드 하단에 붉은색 텍스트로 구체적인 원인을 명시한다.
